# Build offline PWA for forage

## Design

### What it is

A Progressive Web App generated by `forage export --pwa -o ./pwa/`, producing a directory of static files (HTML, JS, CSS, manifest, service worker, icons). Installable on mobile, works offline, supports full CRUD on books with manual diff-based export for syncing back to the CLI.

### Data layer

- **Initial load:** Book and bookseller data embedded as JSON in the HTML (same as today's export)
- **Persistence:** Written to IndexedDB on first load. Subsequent visits read from IndexedDB.
- **Change tracking:** Every mutation (create/update/delete) is logged to a `changes` store in IndexedDB with timestamp and operation type
- **Export to CLI:** "Download Changes" exports the changelog as JSON. CLI applies the operations via `forage import --changes`
- **Re-import from CLI:** Running `forage export --pwa` again produces fresh embedded data. PWA detects newer data and offers to replace, clearing the changelog.

### CLI changes

- Add `--pwa` flag to `forage export`
- When set, output a directory instead of a single HTML file:
  ```
  pwa/
    index.html        # app shell + embedded data
    app.js            # all app logic
    style.css         # styles
    manifest.json     # PWA manifest
    sw.js             # service worker (cache-first)
    icon-192.png      # generated or placeholder
    icon-512.png
  ```
- Add `forage import --changes` to apply PWA changelog diffs to the master DB

### PWA features

- **Installable:** manifest.json with app name, icons, theme color
- **Offline:** Service worker caches all assets on first visit
- **Full CRUD:** Add books, edit all fields (title, author, status, rating, tags, notes), delete books
- **Open Library lookup:** Add-book form has a "Look up" button that queries openlibrary.org/search.json directly (CORS-friendly, no key). Shows canonical title, author, first publish year. User confirms before saving.
- **UI:** Evolve the current card layout — same warm palette, add edit modals/inline editing, mobile-optimized touch targets
- **Search/filter/sort:** Same as current export
- **Sync:** "Download Changes" button exports changelog as JSON diffs

### What's NOT included (YAGNI)

- No server component
- No automatic sync
- No SQLite WASM
- No user accounts or auth
- No book cover images
